### [MEDIUM] Config example uses hardcoded string instead of `binaryName` variable

**File:** `cmd/config.go:38-42`
**Issue:** The `Example` field hardcodes `"changie"` as a string literal:
```go
Example: `  # Validate default config file
  changie config validate
  ...`
```
PLAN.md AD-3 explicitly decided to use `fmt.Sprintf` with `binaryName` to ensure the example is always correct regardless of binary name. This deviation means if `binaryName` were ever changed (e.g., a fork), the example would show the wrong binary name. Other commands like `cmd/completion.go:16-26` use `fmt.Sprintf(... binaryName ...)` for their examples.
**Fix:** Move the Example assignment to `init()` where `binaryName` is resolved:
```go
func init() {
    configValidateCmd.Example = fmt.Sprintf(`  # Validate default config file
  %s config validate

  # Validate specific config file
  %s config validate --file /path/to/config.yaml`, binaryName, binaryName)
    // ...
}
```
